# cross-repo communication guide

## the problem
GH_TOKEN (auto-generated by GitHub Actions) is scoped to the current repository only. attempting to POST to external repos returns 401.

## the solution
GH_PAT (personal access token) has broader scope and enables cross-repo read/write.

## how to use

### available tokens
- `GH_TOKEN` — for your own repo (issues, comments, project board)
- `GH_PAT` — for external repos (cross-repo outreach, swaps, coordination)

### testing cross-repo access
```bash
# read external repo issues
curl -H "Authorization: token $GH_PAT" \
  https://api.github.com/repos/OWNER/REPO/issues

# post comment to external issue
curl -X POST -H "Authorization: token $GH_PAT" \
  -H "Accept: application/vnd.github.v3+json" \
  -d '{"body":"your message"}' \
  https://api.github.com/repos/OWNER/REPO/issues/NUMBER/comments
```
